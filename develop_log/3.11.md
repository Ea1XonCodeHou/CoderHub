# 3.11 开发日志 - 忘记密码 / 邮箱找回方案设计

## 一、业务背景

用户忘记密码后，需要通过已绑定的邮箱进行身份验证并重置密码。
个人项目复杂度，采用最简方案：**邮箱验证码 → 重置密码**，不做链接跳转。

---

## 二、你需要准备什么

| 序号 | 准备项 | 说明 |
|------|--------|------|
| 1 | **一个邮箱账号** | 用于发送验证码的"系统邮箱"（推荐 QQ邮箱 或 163邮箱） |
| 2 | **开启 SMTP 服务** | 在邮箱设置中开启 POP3/SMTP，获取**授权码**（不是登录密码） |
| 3 | **Maven 依赖** | `spring-boot-starter-mail`（项目当前未引入） |
| 4 | **配置文件** | 在 `application.yml` 添加邮箱 SMTP 配置 |
| 5 | **确保用户表有邮箱** | ✅ 已有 `email` 字段 + 唯一索引 idx_email |

### 推荐：QQ邮箱 SMTP 配置获取步骤

1. 登录 QQ邮箱 → 设置 → 账户 → POP3/SMTP服务 → 开启
2. 按提示发短信验证 → 获得 **16位授权码**（保存好，只显示一次）
3. SMTP 地址：`smtp.qq.com`，端口：`465`（SSL）或 `587`（TLS）

---

## 三、技术方案

### 3.1 整体流程（两步式）

```
用户输入邮箱 → 后端校验邮箱存在 → 发送6位验证码到邮箱 → Redis存验证码(5min TTL)
     ↓
用户输入验证码 + 新密码 → 后端校验验证码 → 更新密码 → 返回成功
```

### 3.2 接口设计（2个接口）

| 接口 | 方法 | 路径 | 参数 | 说明 |
|------|------|------|------|------|
| 发送验证码 | POST | `/user/password/send-code` | `{ email }` | 校验邮箱存在 → 发验证码 |
| 重置密码 | POST | `/user/password/reset` | `{ email, code, newPassword }` | 校验验证码 → 更新密码 |

### 3.3 验证码存储

- Redis Key：`captcha:{email}`（已有常量 `RedisConstant.CAPTCHA`）
- Value：6位随机数字
- TTL：5分钟（已有常量 `RedisConstant.EXPIRE_5_MINUTES`）
- 防刷：同一邮箱 60秒内不可重复发送（用 Redis key TTL 判断剩余时间）

## 四、实现路线（编码顺序）

### Step 1：引入依赖 + 配置

**pom.xml**（coderhub-server）:
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-mail</artifactId>
</dependency>
```

**application.yml** 新增：
```yaml
spring:
  mail:
    host: smtp.qq.com
    port: 465
    username: your_email@qq.com
    password: 你的16位授权码
    properties:
      mail.smtp.ssl.enable: true
      mail.smtp.auth: true
    default-encoding: UTF-8
```

### Step 2：创建 MailService（发邮件工具类）

**路径**：`service/MailService.java`

核心方法：
```java
public void sendVerificationCode(String toEmail, String code) {
    SimpleMailMessage message = new SimpleMailMessage();
    message.setFrom(fromEmail);      // 配置中的发件人
    message.setTo(toEmail);
    message.setSubject("CoderHub 密码重置验证码");
    message.setText("您的验证码是：" + code + "，5分钟内有效。如非本人操作请忽略。");
    mailSender.send(message);
}
```

依赖注入 `JavaMailSender`（Spring Boot 自动配置）。

### Step 3：UserController 添加2个接口

**发送验证码接口** `/user/password/send-code`：
1. 根据 email 查询用户是否存在 → 不存在返回"该邮箱未注册"
2. 检查 Redis `captcha:{email}` 是否存在 → 存在说明60秒内已发过，返回"请稍后再试"
3. 生成6位随机验证码 → 存入 Redis（TTL 5分钟）
4. 调用 MailService 发送邮件
5. 返回成功

**重置密码接口** `/user/password/reset`：
1. 从 Redis 获取 `captcha:{email}` → 不存在返回"验证码已过期"
2. 比对验证码 → 不匹配返回"验证码错误"
3. 更新用户密码（MD5 或后续升级为 BCrypt）
4. 删除 Redis 中的验证码（一次性）
5. 返回成功

### Step 4：前端页面

在登录页添加"忘记密码？"链接，跳转到密码重置页面：
- 第一步：输入邮箱 + 发送验证码按钮（60秒倒计时）
- 第二步：输入验证码 + 新密码 + 确认密码
- 提交后跳转回登录页

## 五、安全注意事项

| 项目 | 措施 |
|------|------|
| 防刷 | 同一邮箱 60秒限频 + 同一 IP 可加 rate limit |
| 验证码安全 | 6位数字 + 5分钟过期 + 验证成功后立即删除 |
| 密码安全 | 当前项目用 MD5（⚠️ 建议后续升级 BCrypt） |
| 邮箱泄露 | 发送验证码接口不应暴露"该邮箱未注册"（可统一返回"如邮箱已注册将收到验证码"） |
| 授权码保密 | SMTP 授权码放在 application-prod.yml，不要提交到 Git |
